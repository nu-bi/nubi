OSS RLS RS256 Nubi Embed — Per-Tenant Isolation Demo

This page proves the Nubi embed story end-to-end. Two tenants share the same dashboard and the same Nubi backend — but each sees only their own rows. Isolation is enforced server-side via JWT policies.tenant_id claims; neither the embed component nor the browser can alter or bypass it.

Per-Tenant RLS Isolation — What the backend enforces

Acme Corp token
  • policies: {"tenant_id": "acme-corp"}
  • scope: ["read:*"]
  • embed_origin: http://localhost:8080
  • Backend SQL: … WHERE tenant_id='acme-corp'

  • policies: {"tenant_id": "globex-inc"}
  • scope: ["read:*"]

How It Works

Browser tenant switcher ──▶ host /api/embed-token mints RS256 JWT (policies.tenant_id = X) ──▶
Acme Corp JWT or Globex Inc JWT ──▶ nubi-dashboard component ──▶
POST /api/v1/query Nubi backend verifies RS256 sig + embed_origin + exp
injects WHERE tenant_id='acme-corp' as server-side RLS ──▶ DuckDB/PG
◀── only tenant-scoped rows returned to the component
